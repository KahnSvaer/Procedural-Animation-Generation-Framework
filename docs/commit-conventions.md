# Commit Conventions

This project uses a custom commit convention designed for animation, rigging, procedural animation, and research workflows.

## Types

### feat

New capability or feature.

Examples:

* feat: add procedural tail controller
* feat: export animation pipeline to Unity

### fix

Fix broken functionality.

Examples:

* fix: correct bone orientation
* fix: resolve export crash

### rig

Rigging work.

Examples:

* rig: create fish spine chain
* rig: add fin control bones

### anim

Animation work.

Examples:

* anim: add swim cycle
* anim: create idle fin motion

### asset

Asset changes.

Examples:

* asset: add generated fish mesh
* asset: update fish textures

### exp

Experiments, investigations, and prototypes.

Examples:

* exp: evaluate automatic fin detection
* exp: compare tail bone configurations

### docs

Documentation changes.

Examples:

* docs: document rigging architecture
* docs: add development notes

### refactor

Code restructuring without changing functionality.

Examples:

* refactor: simplify export pipeline
* refactor: reorganize animation modules

### test

Validation and testing.

Examples:

* test: verify Unity import workflow
* test: validate swim cycle playback

### repo

Repository configuration, project structure, infrastructure, CI/CD, and GitHub configuration.

Examples:

* repo: initialize repository structure
* repo: add git lfs configuration
* repo: configure GitHub Actions
